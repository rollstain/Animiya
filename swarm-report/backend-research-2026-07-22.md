# Research: выбор бекенда и источников данных для Animiya

**Дата:** 2026-07-22
**Профиль:** Research (только исследование, без перехода к коду — по решению пользователя)
**Проект:** Animiya — свой KMP-клиент для просмотра аниме (Compose Multiplatform + Koin + Decompose + Ktor + SQLDelight) + собственный backend-прокси (Kotlin/Spring Boot)
**Метод:** 6 параллельных исследовательских агентов с доступом в веб. Все факты перепроверялись по официальной документации, живым HTTP-запросам и исходникам сторонних SDK. Непроверенное помечено явно.

> ⚠️ Оговорка о консилиуме из CLAUDE.md: штатные субагенты (`java-architect`, `api-designer`, `security-kotlin` и т.д.) не имеют доступа в веб и работают по коду, которого у нас нет (пустой скелет проекта). Для research по внешним API это дало бы галлюцинации. Поэтому применён тот же принцип «параллельная экспертиза», но агентами с реальным веб-доступом.

---

## 0. Главные выводы (TL;DR)

1. **Полностью легального пути к лицензионному аниме-видео через публичное API — нет.** Ни Crunchyroll, ни Кинопоиск/Иви/Okko/Wink не дают партнёрского API для встраивания видео. Легальный контент = прямой лицензионный договор или B2B уровня Wakanim/ADN. Это дорого и медленно.
2. **Kodik/Alloha — красная зона.** В 2026 антипиратская компания F6 (экс-Group-IB) публично назвала Alloha (61% пиратских сайтов) и Kodik (9%) ведущей инфраструктурой нелегальных ресурсов и заявила о передаче данных в правоохранительные органы. Структурный прецедент последствий: **Anixart уже удалён из Google Play** (полная версия — только сайдлоад APK).
3. **Оба «якорных» русских источника под давлением РКН.** Shikimori заблокирован (`.one`→`.io`, вычистка контента под требования регулятора). AniLibria прошла ребрендинг в **AniLiberty** и тоже под блокировками. Это не разовые инциденты, а тренд — закладывать в архитектуру как постоянный риск.
4. **Рекомендуемая раскладка источников** (детали в §7):
   - **Видео:** AniLiberty (`anilibria.top/api/v1`, открытый HLS API) — низкий технический риск, но это фандаб без лицензии (юридический риск остаётся).
   - **Каталог/метаданные:** AniList (богатый GraphQL) как первичный + **Shikimori — обязателен ради нативного русского текста**.
   - **Синхронизация списков (часть MVP):** OAuth к Shikimori + AniList.
   - **Склейка источников:** через MAL id как общий якорь (есть у всех).
5. **Свой Spring Boot прокси — оправдан и необходим**, не «для галочки»: абстракция домена под будущие блокировки/миграции, кеш под жёсткие rate-limit (AniList 30 rpm degraded, Shikimori 5 rps/90 rpm), сокрытие ключей, буфер от нестабильности апстрима, точка для своих фич.

**Решение, которое нужно от тебя:** позиция по видео-источнику (см. §8) — принимаем юридический риск AniLiberty ради быстрого старта, или сразу целимся в лицензионный путь. От этого зависит вся продуктовая стратегия и публикация в сторах.

## 1. Видео-источники

### 1.1 AniLiberty (бывш. AniLibria) — рекомендуемый источник видео

- **Актуальный API:** `https://anilibria.top/api/v1` (зеркало `https://api.anilibria.app/api/v1`). Старые `api.anilibria.tv/v2,v3` — **отключены, HTTP 410 Gone**.
- **Swagger:** `https://anilibria.top/api/docs/v1` (SPA). Сырая OpenAPI: `https://anilibria.top/storage/api/docs/v1?aniliberty-api-v1-docs.json` (~289 KB).
- **Формат видео (подтверждено живым запросом `/anime/releases/7443`):** каждый эпизод содержит плоские `hls_480`/`hls_720`/`hls_1080` — готовые абсолютные `.m3u8` URL (могут быть `null`). Только HLS, mp4 нет.
- **CDN:** `cache.libria.fun` за Cloudflare. На манифест Referer/UA-ограничений не обнаружено (`.ts`-сегменты отдельно не тестировались).
- **Таймкоды опенинга/эндинга:** есть — `opening{start,stop}` / `ending{start,stop}` в секундах (заполнены не для всех серий). Это снижает потребность в AniSkip.
- **Читать каталог/релизы/жанры/расписание** можно анонимно. Есть пользовательские фичи через API: избранное (`accounts/users/me/favorites/releases`), прогресс просмотра (`.../views/timecodes`) — но точный механизм авторизации для v1 не подтверждён (Swagger — SPA).
- **Расписание:** дни недели 1(Пн)…7(Вс) — отличается от старого v3 (0…6). Учитывать при миграции референсного кода.
- **Торренты:** свой трекер (`anime/torrents`, `/release/{id}`, `/{hash}/file`). RSS для v1 не подтверждён (был в v3).
- **Rate limits / обязательный UA / CORS — нигде не задокументированы.** Ещё один довод за прокси (закрыть неопределённость централизованно).
- **Стабильность:** подтверждённые жалобы на 500/504 и краткие CDN-инциденты (status.anilibria.top фиксирует). Крупный breaking change: полный вывод v2/v3 + ребрендинг + смена домена. Версия «v1» переиспользована после ребрендинга (был старый v1 2016 → v2 → v3 → новый v1).
- **Правовой статус:** явного ToS для API нет. **Важная поправка:** AniLiberty — продолжение **фандаб**-направления (любительский перевод **без лицензии** правообладателей), дисклеймер «для домашнего ознакомительного просмотра». Лицензированная **AniLibria Studio** (ООО «ТРИПЛЭЙ ПЛЮС») — отдельное юрлицо и **API для сторонних приложений не даёт**. Домен `anilibria.tv` блокировался Мосгорсудом ещё в 2019 (иск «Синема Галэкси»).
- **Вердикт:** технически — лучший доступный источник видео (открытый, документированный, стабильный контракт, готовые HLS + таймкоды). Юридически — нелицензированный фандаб; риск не снимается, а смещается с «непрозрачный агрегатор» на «прозрачный, но нелицензированный».

### 1.2 Kodik / Alloha и прочие балансеры — НЕ рекомендуется

- **Модель:** B2B видеобалансеры, встраиваемый iframe-плеер, токен выдаётся вручную по письму (нет публичной оферты). Поиск по внешним id (shikimori_id, kinopoisk_id, imdb_id; mal_id у Kodik не подтверждён).
- **Правовой статус — критично:** отчёт F6 (2026, Habr): **Alloha — №1 инфраструктура пиратских сайтов (61%)**, Rewall (42%), Lumex (11%), **Kodik (9%)**, HDVB (7%); заявлена передача данных в правоохранительные органы.
- **Продуктовый риск (сторы):** Apple 5.2.2/5.2.3 требуют разрешение от первоисточника прав «upon request»; Google Play IP Policy требует письменную лицензию — у балансер-модели такого документа структурно нет. Реальный прецедент — **Anixart вылетел из Google Play**.
- **Технический риск:** неформальный отзываемый токен, частая смена доменов (kodik.cc→kodik-api.com→kodikplayer.com в 2026), полная зависимость от чужого сервиса без договора. Прецедент 2019: Kodik уже терял половину каталога под давлением.
- **Реверс чужих APK Anixart/AniNet ради их токенов — вне рассмотрения** (незаконно, токен отзовут, сборку с ним нельзя публиковать).

### 1.3 Прочие RU-источники видео

- **Anime365 / SmotretAnime:** официального публичного API нет (только реверс-обёртки), видео по платной подписке (~2500 ₽/год). Не подходит без официального партнёрства.
- **Animevost / Sovetromantica / Animedia:** нет публичного API, серая зона.
- **Jut.su, AnimeGO:** заблокированы РКН (февраль 2024).
- **Легальные площадки РФ (Crunchyroll, Кинопоиск, Иви, Okko, Wink):** публичного партнёрского API для встраивания видео/каталога — нет. Максимум — deep-link/redirect в их официальные приложения.

## 2. Каталог и метаданные

| Критерий | AniList | MAL API v2 | Jikan | Kitsu | Shikimori |
|---|---|---|---|---|---|
| Тип | GraphQL | REST | REST (обёртка MAL) | JSON:API | REST v1/v2 (deprecated) + GraphQL |
| Чтение без токена | ✅ | ✅ | ✅ (записи нет) | ✅ | ✅ (но обязателен кастомный User-Agent) |
| Rate limit сейчас | **30/min (degraded)**, номинал 90 | не публикуется | ~3/с (community) | не подтв. | **5/с, 90/мин** |
| При перегрузке | 429 + возможна ручная блокировка IP | — | наблюдалась 500 от апстрима | — | бан IP при плохом UA |
| Запись прогресса | ✅ SaveMediaListEntry | ✅ PATCH my_list_status | ❌ | ✅ library-entries | ✅ (через GraphQL; REST v1 deprecated) |
| Родной русский текст | ❌ | ❌ | ❌ | ❌ | **✅ `russian` + RU description** |
| Маппинг на MAL | ✅ `idMal` | (сам MAL) | (сам MAL) | ✅ `mappings` | ✅ `myanimelist_id` |
| Коммерческое исп. | запрета нет; нужен 18+ фильтр для сторов | **ToS: personal/noncommercial, запрет агрегации** | наследует риск MAL | ToS не подтв. | явно не подтв.; жёсткий UA |
| Ключевой риск | многолетний degraded 90→30 | скудная докум. лимитов | зависит от разметки MAL | **maintenance-only** | смена домена .one→.io |

**Итог по метаданным:**
- **AniList — первичный каталог:** богатейший граф (relations типизированы, теги со спойлер-флагами, airingSchedule, recommendations, trailer, characters+staff, studios), нет MAL-подобного запрета на коммерцию, встроенный `idMal`. Ограничение — degraded 30 rpm, лечится агрессивным кешем на прокси.
- **Shikimori — обязателен, а не опция:** единственный источник с нативным русским `russian`/`description`. Стабильнее по лимиту (5/с, 90/мин), но обязателен кастомный User-Agent (иначе бан IP), домен — `shikimori.io`, не хардкодить.
- **MAL API v2 — только как связующий ключ (id):** ToS буквально «personal, noncommercial use» + запрет агрегации «for use elsewhere». Прямая коллизия с моделью прокси-каталога. Не первичный источник.
- **Jikan — максимум read-only fallback для некоммерческого MVP** (наследует риск MAL, записи нет, наблюдалась 500).
- **Kitsu — низкий приоритет:** данных сверх AniList не даёт; ценность только в `mappings`. Сервис в maintenance-only, домен io↔app нестабилен. Канонический base — `kitsu.app`, но конфигурируемый.

Все проверены через GitHub API (звёзды, дата push, лицензия).

Ближайший аналог по функционалу. ⚠️ AGPL — **изучать архитектуру, не копировать код** (иначе продукт обязан стать AGPL).

3. **[mihonapp/mihon](https://github.com/mihonapp/mihon)** (22k★) + **[aniyomiorg/aniyomi](https://github.com/aniyomiorg/aniyomi)** (7.5k★), Apache-2.0. Эталон extension/source-модели: разделение «трекер метаданных» ↔ «источник видео» через версионированный контракт адаптера. Прямо переносимо на дизайн нашего backend-агрегатора (`ShikimoriAdapter`, `AniLibriaAdapter`).
4. **[anilibria/anilibria-app](https://github.com/anilibria/anilibria-app)** (245★, Kotlin, GPL-3.0) + **[anilibria/alice](https://github.com/anilibria/alice)** (Go, MIT) — официальный клиент (эталон интеграции именно с AniLibria API) + официальный кеш-прокси (подтверждает: даже сам AniLibria держит кеширующий слой перед своим API — наш прокси идёт тем же путём).
5. **[gnoemes/Shimori](https://github.com/gnoemes/Shimori)** (45★, Apache-2.0, заморожен 2025-05) — единственный пример **SQLDelight-кеша каталога Shikimori в KMP**. Ценен точечно для модели данных, не как живой проект.

Доп.: **[alvr/katana](https://github.com/alvr/katana)** (Apache-2.0, свежий, маленький Compose MP клиент AniList) — удобный starter по структуре Koin/Ktor-модулей ближе к нашему масштабу.

⚠️ **DMCA-сигнал:** Saikou и основной форк Dantotsu (rebelonion) на 2026 **заблокированы GitHub по DMCA** (жалобы Corsearch/Crunchyroll) — прямое следствие extension-модели на нелегальные видео-источники. Ещё один довод против этого пути.

## 5. Технические практики для стека

- **Видеоплеер HLS в Compose MP:** `open-ani/mediamp` (см. выше). Альтернативы (ber4444/kmp-videos, sakuro) — слишком свежие/experimental.
- **Офлайн-скачивание HLS:** готовой кросс-платформенной KMP-библиотеки нет. Android — Media3 `DownloadService`, iOS — `AVAssetDownloadTask`, склейка через expect/actual. Даже animeko написал это вручную.
- **SQLDelight офлайн-кеш каталога:** миграции `.sqm` (частый баг — забыть добавить при смене схемы), `coroutines-extensions` для Flow, platform-специфичные драйверы (WAL-различия). Паттерн `updated_at`/`etag`/`sync_version` + upsert по внешнему id → фоновая синхронизация вместо live-проксирования.
- **Secure storage OAuth-токенов:** `Liftric/KVault` (Keychain/Keystore из коробки) для токенов + `russhwolf/multiplatform-settings` для несекретных настроек.
- **Кеш rate-limited API на бекенде:** Spring Cache + Redis/Caffeine (L2) + `@Scheduled`/Quartz для инкрементальной синхронизации каталога в свою БД. Модель Jikan/alice: не проксировать запрос-в-запрос, а держать свой снапшот.

## 6. Продуктовый срез (сравнение приложений-конкурентов)

| Приложение | Модель | Ключевые фичи |
|---|---|---|
| **AniLibria/AniLiberty** | Своя озвучка + каталог | Избранное с цветовым прогрессом, комментарии, новости, торренты, **настраиваемый endpoint API + статик-сервер** (подтверждает развязку клиент↔бекенд контрактом — наша схема) |
| **Anixart** | Агрегатор + внешние балансеры | ~7100 тайтлов, рекомендации по оценкам, публичные/личные коллекции, ТОПы, профили, мультиозвучка, статистика. Google Play — только Lite без видео |
| **AniNet (GaglArt)** | Свой каталог + агрегация видео | 15000+ тайтлов, календарь, ИИ-рекомендации, **импорт списков из Shikimori/MAL**, Chromecast, офлайн, Premium+реклама |
| **Shikimori App** | Клиент API Shikimori | Списки со «продолжить с места», расширенный поиск, календарь онгоингов, выбор плеера, соцфункции |

**Продуктовые выводы:**
1. **Импорт/синхронизация списков (Shikimori, MAL) — ожидание аудитории по умолчанию** (есть у AniNet, Shikimori App). Значит OAuth к Shikimori/AniList — часть MVP, не «на потом».
2. **Настраиваемый endpoint у AniLibria** — подтверждение, что наша прокси-архитектура (клиент не знает источник) правильна.
3. Коллекции, профили, статистика, мультиозвучка (выбор команды перевода), push на новые серии — базовый набор конкурентоспособности.

## 7. Рекомендуемая целевая архитектура источников

```
                    ┌─────────────────────────────────────────┐
   KMP-клиент  ───▶ │      Spring Boot backend-прокси          │
 (Compose MP,       │  единый контракт, кеш, свои фичи, ключи   │
  Decompose,        └───────────────┬─────────────────────────┘
  Ktor, SQLDelight)                 │  адаптер-на-источник
                                    ▼
      ┌──────────────┬──────────────┬──────────────┬────────────────┐
      ▼              ▼              ▼              ▼                ▼
  AniList        Shikimori      AniLiberty     Fribb/anime-     AnimeThemes /
 (метадата,     (RU-текст,     (видео HLS,     lists (id-       AniSkip
  граф, id,      user_rates,    таймкоды,       маппинг MAL↔     (доп. фичи)
  airing)        избранное)     торренты)       AniList↔...)
      │              │
      └──── OAuth ────┘  (синхронизация списков — MVP)

  Склейка всех записей по MAL id как общему якорю.
```

**Роли:**
- **AniList** — «мировая» метадата: расписание, теги, франшизы, рекомендации, изображения, `idMal`.
- **Shikimori** — RU-название и RU-описание (по `myanimelist_id`), OAuth-синхронизация списков/прогресса.
- **AniLiberty** — видео (HLS), таймкоды OP/ED, торренты.
- **Fribb/anime-lists** — основа таблицы кросс-маппинга id.
- **AnimeThemes/AniSkip/TMDB** — опциональные улучшения.

**Почему свой прокси обязателен (а не прямые вызовы из клиента):**
1. Абстракция домена — под неизбежные блокировки/миграции (RKN бьёт по обоим RU-источникам).
2. Кеш под жёсткие лимиты (AniList 30 rpm degraded с угрозой ручного бана IP; Shikimori 5/с) — иначе при росте пользователей клиент упрётся в лимит и получит бан по IP на всех.
3. Единый User-Agent для Shikimori (обязателен, иначе бан).
4. Сокрытие ключей (TMDB и др.) вне APK.
5. Буфер от нестабильности апстрима (AniLibria 500/504).
6. Место для своих фич: единый список избранного/прогресса поверх нескольких трекеров, push-уведомления, нормализация моделей.

## 8. Открытые вопросы и решения, нужные до старта

1. **[РЕШЕНИЕ ЗА ТОБОЙ] Позиция по видео.** AniLiberty даёт технически чистый путь, но это нелицензированный фандаб — риск претензий/блокировок и проблем с публикацией в сторах остаётся. Варианты:
   - (a) Принять риск, стартовать на AniLiberty, распространять вне Google Play при необходимости (как Anixart);
   - (b) Целиться в лицензионный путь (Wakanim/ADN/прямой договор) — дорого, медленно, но предъявимо модераторам сторов;
   - (c) Гибрид: MVP как «трекер + агрегатор ссылок» с deep-link, видео — опционально.
2. **Юридическая проверка Shikimori:** явной лицензии на коммерческое использование API нет — до продакшна написать администрации (клуб shikimori.io/clubs/950-shikimori-api).
3. **MAL — только id-ключ**, не первичный источник (ToS-коллизия).
4. **Хостинг прокси и TMDB:** если РФ-хостинг и используем TMDB — нужен не-российский egress.
5. **Лицензии OSS:** animeko (AGPL) и клиенты AniLibria (GPL) — изучать, не копировать код. Строить на Apache/MIT-компонентах (mediamp, mihon-паттерны, KVault, multiplatform-settings, Fribb).
6. **Уточнить вручную (SPA-доки не парсились):** механизм авторизации AniLiberty v1; GraphQL-мутации user_rates Shikimori; точные rate-limit MAL v2/Kitsu; лицензия Fribb/anime-lists.
